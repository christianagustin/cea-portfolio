The Problem

Expanding a lead generation system from one market to another sounds simple. In practice, every layer of the system has market-specific logic that must be rebuilt or adapted.

Google Maps searches in English miss locally-owned Hong Kong businesses that operate exclusively under Chinese names. Phone validation rules are completely different. The business registry is a different government system with a different API. Name-match scoring must handle Traditional Chinese characters.

None of these differences can be assumed away. Each one was a silent failure mode that would have produced empty or inaccurate data without explicit handling.

A system that works in one market is a proof of concept. A system that works in two markets is an architecture.

How It Works

The five-workflow structure was kept intact. Every market-specific layer was rebuilt rather than patched.

Lead DiscoveryGoogle Maps queries submitted in Traditional Chinese. A cleaning company in Hong Kong with no English-language listing simply does not appear in an English-language search. Switching the query language expanded the addressable lead pool significantly.
Phone ValidationPhone normalization rules updated for the +852 country code. The regex pattern was rewritten for HK-format numbers. A phone that passes Singapore validation fails HK validation and vice versa.
Registry LookupThe national business registry scraping approach used in Singapore was replaced with a direct API call to the Hong Kong Companies Registry open data endpoint. The API returns company name, registration number, status, and address in JSON. No scraping, no login, no cost.
Name ScoringThe confidence scoring algorithm that matches a Maps result against registry results was updated to handle Traditional Chinese character strings, including partial matches where only the Chinese portion of a bilingual name is present in one source.
Client Cross-ReferenceSame logic as the Singapore variant, but references the HK client master sheet and matches on HK-format phone numbers.

The HK Companies Registry API

The Hong Kong Companies Registry publishes a free open data API that allows any caller to search registered companies by name or registration number without authentication. The response is structured JSON and returns registration status directly.

Outcome & Impact

  • A second market running in parallel from the same automation infrastructure, with no additional engineering overhead once adaptations were built.
  • The HK registry API is a cleaner data source than the SG equivalent — authoritative, always current, and free.
  • The architecture proved scalable. A third or fourth market variant requires adapting five layers. The core pattern does not change.
  • Both markets' sales teams start with enriched, validated, deduplicated lead lists simultaneously.
